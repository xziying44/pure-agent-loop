# Skill 系统提示词集成设计

## 概述

将 Skill 工具的使用说明集成到内置系统提示词中，引导智能体主动加载技能，而无需用户在自定义提示词中指定。

## 目标

1. **智能体自主加载技能**：通过系统提示词引导，智能体能在任务匹配时主动调用 skill 工具
2. **对用户无感**：加载技能不需要出现在 todo 任务清单中
3. **保留调试能力**：仍然产出 ACTION/OBSERVATION 事件，方便开发者调试

## 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 触发机制 | 纯提示词引导 | 简单、不增加框架复杂度 |
| 不列入 todo 的实现 | 提示词约束 | LLM 能遵循明确指令，保持框架通用性 |
| 提示词位置 | "高效工具使用"之后、"任务管理"之前 | Skill 本质是工具使用策略的补充 |
| 详细程度 | 详细版 + 精简其他部分 | Skill 是核心功能，值得详细说明 |

## 变更内容

### 1. 新增：技能加载章节

位置：`## 高效工具使用` 之后

```markdown
## 技能加载
你可以通过 skill 工具加载专业技能，获取特定任务的详细指导。

### 何时加载技能：
- 任务涉及专业领域（搜索策略、代码审查、数据分析等）
- 不确定最佳实践或工具使用方式
- 需要遵循特定流程或规范

### 加载原则：
- 在开始具体工作前完成加载，根据技能指导调整后续策略
- 技能加载是基础准备工作，无需列入任务清单
```

### 2. 精简：高效工具使用章节

**原版（约 180 字）** → **精简版（约 80 字）**

```markdown
## 高效工具使用
当多个工具调用**相互独立**时，应在一次回复中并行返回，而非逐个调用等待结果。但如果后续调用依赖前一个结果，则必须顺序执行。
```

### 3. 精简：任务管理的"调整规范"部分

**原版（约 200 字）** → **精简版（约 100 字）**

```markdown
### 动态调整任务列表
任务列表应根据实际情况灵活调整：
- 获得新信息时追加子任务
- 工具调用失败时调整策略
- 接近资源限制时优先保留核心任务

调整前先在回复中说明原因，避免无理由删除或忽略新信息。
```
